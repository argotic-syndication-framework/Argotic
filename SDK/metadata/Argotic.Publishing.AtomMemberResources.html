Remarks

The AtomMemberResources class implements the app:collection element of the Atom Publishing Protocol.

The AtomMemberResources describes a AtomFeed. The AtomMemberResources must specify a Title and Uri.

The AtomMemberResources may contain any number of AtomAcceptedMediaRange entities, indicating the types of representations accepted by the AtomMemberResources. The order of such elements is not significant. Additionally, the AtomMemberResources may contain any number of AtomCategoryDocument.

The AtomMemberResources may appear as a child of an AtomFeed or AtomSource element in an AtomFeed document. Its content identifies a collection by which new entries can be added to appear in the feed.

Accepts

Gets a list of media ranges that are accepted by this collection.

Declaration
public Collection<AtomAcceptedMediaRange> Accepts { get; }
Property Value
Type Description
System.Collections.ObjectModel.Collection<AtomAcceptedMediaRange>

A System.Collections.ObjectModel.Collection<T> of AtomAcceptedMediaRange objects that represent a list of media ranges that this collection will accept from clients.
Remarks

A value of application/atom+xml;type=entry may appear in any AtomAcceptedMediaRange list of media ranges and indicates that AtomEntry can be added to the AtomMemberResources. If no AtomAcceptedMediaRange is present, clients should treat this as equivalent to an AtomAcceptedMediaRange with the content application/atom+xml;type=entry. The AtomAcceptedMediaRange class exposes a static string property named AtomEntryMediaRange that can be used to assign the application/atom+xml;type=entry media range value.

If one AtomAcceptedMediaRange exists and is empty, clients should assume that the AtomMemberResources does not support the creation of new Entries.

System.Collections.ObjectModel.Collection<AtomCategoryDocument>

A System.Collections.ObjectModel.Collection<T> of AtomCategoryDocument objects that represent a list of categories that can be applied to members of this collection.
Remarks

The server may reject attempts to create or store members whose categories are not present in its categories list. A AtomMemberResources that indicates the category set is open should not reject otherwise acceptable members whose categories are not in its categories list. The AtomLink that is returned has a Relation of edit-media. The value of edit-media specifies that the value of the href attribute is an IRI that can be used to modify a media resource associated with an AtomEntry.

An AtomEntry may contain zero or more edit-media link relations. All edit-media link relations in the same AtomEntry reference the same Resource. If a client encounters multiple edit-media link relations in an AtomEntry then it should choose a link based on the client preferences for ContentType and ContentLanguage. If a client encounters multiple edit-media link relations in an AtomEntry and has no preference based on the ContentType and ContentLanguage then the client should pick the first edit-media link relation in document order.

SlugDecode(String)

Decodes the percent-encoded value of the UTF-8 encoding of a character sequence that represents a Slug HTTP entity-header value.

Declaration
public static string SlugDecode(string slug)
Parameters
Type Name Description
System.String slug

The percent-encoded value of the UTF-8 encoding of a character sequence to be included in a Uri.
Returns
Type Description
System.String
Remarks

Slug is an HTTP entity-header whose presence in a POST to a AtomMemberResources constitutes a request by the client to use the header's value as part of any URIs that would normally be used to retrieve the to-be-created Entry or Media Resources.

Servers may use the value of the Slug header when creating the Member URI of the newly created Resource, for instance, by using some or all of the words in the value for the last URI segment. Servers may also use the value when creating the AtomId, or as the Title of a Media Link Entry.

Servers may choose to ignore the Slug entity-header. Servers may alter the header value before using it. For instance, a server might filter out some characters or replace accented letters with non-accented ones, replace spaces with underscores, change case, and so on.
